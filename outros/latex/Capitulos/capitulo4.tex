\chapter{Proposta}
Este capítulo apresenta a proposta de desenvolvimento de um sistema \textit{web}, capaz de auxiliar na criação e divulgação de conteúdos que permitem a interação com o usuário que o acessa, resultando em uma imagem personalizada, a partir dos dados do \textit{Facebook} do usuário. Para facilitar a compreensão utilizaremos o nome de \textbf{aplicativo} para definir este tipo de conteúdo.


\section{VISÃO GERAL}
A proposta desse trabalho é auxiliar na criação, divulgação e monetização de conteúdos dinâmicos. O primeiro passo é a criação do conteúdo, onde será desenvolvido uma ferramenta para auxiliar sua criação, que será descrita em detalhes na seção \ref{gc}. Depois desta parte, o criador de conteúdo irá gerar um  \textit{link} e divulga-lo em sua  \textit{Facebook Page}. A Figura \ref{opengraph} mostra como é a publicação de um \textit{link} em uma \textit{Facebook Page}.
Os usuários ao ter acesso a publicação feita pelo criador de conteúdo, irão acessar o site e poderão interagir com o conteúdo, resultando em uma imagem personalizada, como é descrito em detalhes na seção \ref{site}. Após esse passo o usuário terá a opção de compartilhar o seu resultado em seu \textit{Facebook}, da mesma forma que foi feito pelo criador de conteúdo, porém agora o conteúdo que ele irá publicar será personalizado com os seus dados. Esse conteúdo publicado poderá ser acessado por seus amigos e iniciar todo o ciclo novamente, abrindo a possibilidade da criação de um \textit{viral}.

Para isso os sistema deve conter os seguintes requisitos funcionais:
\begin{itemize}
	\item \textbf{RF1} O sistema deve permitir níveis de acesso, conforme o papel especificado. Os papéis são: administrador ou criador de conteúdo.
	\item \textbf{RF2}  Os administradores irá realizar todas as atividades que um criador de conteúdo possa realizar;
	\item \textbf{RF3}  O sistema deve permitir que criadores de conteúdos criem aplicativos;
	\item \textbf{RF4}  O sistema deve permitir que os administradores possam gerenciar todos os usuários;
	\item \textbf{RF5}  O sistema deve permitir que os criadores de conteúdo consigam gerar um link para a divulgação.
\end{itemize}

\section{SITE}\label{site}
O \textit{site} é o sistema \textit{web} que irá interagir com o usuário. Para que essa interação aconteça ele irá ver a publicação do \textit{link} em sua rede social, ao acessar ele irá poder interagir com o sistema. O diagrama de caso de uso geral abaixo mostra uma visão ampla do funcionamento do site.

\begin{figure}[!htb]
	\centering
	\includegraphics[width=1\textwidth]{./Figuras/caso-de-uso-site.png} % <- formatos PNG, JPG e PDF
	\caption[Caso de uso geral do site]{Caso de uso geral do site.}
	\fonte{Autoria própria}
	\label{casodeusogeralsite}
\end{figure}

Para isso o sistema deve conter os seguintes requisitos funcionais:
\begin{itemize}
	\item \textbf{RF6} O sistema deve permitir que os usuários realizem a autenticação a partir do Facebook Login;
	\item \textbf{RF7}  O sistema deve permitir que o usuário interaja com o aplicativo resultando em uma imagem;
	\item \textbf{RF8}  O sistema deve permitir que o usuário compartilhe o seu resultado.
\end{itemize}
